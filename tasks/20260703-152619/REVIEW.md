Assessment against the Goal and Steps:
- Every wired event maps to a distinct handle and fires once at the right
  moment. Correctness spot-checks:
  - Fruit burst (`splat`) is placed in `resolve_slice_pop`, which only runs for
    entities carrying `SlicePop`; bombs skip the pop and explode directly, so a
    bomb never gets a fruit splat. This was the main risk called out in the
    task and it is handled correctly (and documented in a comment).
  - `SfxAssets` is inserted in `setup` (Startup), which runs before the first
    Update where `menu_click` reads it - same lifetime pattern as the existing
    `FruitAssets`, so no missing-resource panic (confirmed by the run).
  - Combo chime pitch `1.0 + (count-2)*0.06` capped at 1.6 is well-formed
    (count=2 -> 1.0, monotonic, bounded); paired with the audio module's
    positive-speed clamp it cannot feed rodio a bad rate.
  - Game-over sting hangs off `OnEnter(GameOver)`, so it covers both bomb death
    and the Escape give-up path - better than hanging it off `on_player_died`.
- The dev-only `wav` feature is the right call: it keeps the library's default
  features untouched (matters for a copy-pastable crate) while making the
  example self-contained. Vorbis stays default-on and the README documents the
  one-line `.ogg` swap.
- Mock assets are committed under a non-ignored `assets/`, generated by a
  stdlib-only reproducible script (no ffmpeg/sox dependency).
- `assets/sounds/README.md` lists every required file with event, character and
  length, satisfying the "drop in without touching code" deliverable.
