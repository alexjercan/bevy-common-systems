# 07_orbit: orb-pickup streak juice (combo scoring, rising pitch + combo sound, floating +N popups)

- STATUS: CLOSED
- PRIORITY: 90
- TAGS: feature,example,historical

Polish pass on `examples/07_orbit.rs`. The base SFX are already wired
(menu/pickup/hurt/level_up/game_over); this task adds the "juice" layer that
`06_fruitninja` has but Orbit Runner lacks, so collecting orbs feels rewarding
and the audio has variation instead of one flat blip. Mirrors the fruitninja
polish tasks 20260703-132210 (floating +N) and 20260703-132214 (combo).

Reuse existing assets: `assets/sounds/combo.wav` already exists (generated by
`scripts/gen-placeholder-sounds.py`) and `PlaySfx` already supports
`.with_speed()` for pitch. No new crates, no new assets.

## Behavior

- A streak forms when orbs are collected in quick succession (a short time
  window, refreshed on each pickup, same idea as fruitninja's `Combo`). The Nth
  orb in the streak is worth more (escalating), and the streak resets when the
  window lapses.
- Each pickup plays `pickup.wav` with its playback speed nudged up as the
  streak grows (`PlaySfx::new(..).with_speed(..)`), so a run of pickups rises
  in pitch. At streak x2+ also ping `combo.wav`.
- Each pickup spawns a floating "+N" text popup at a screen position that
  rises and fades out over ~0.8 s, then despawns (port fruitninja's
  `spawn_floating_text` / `animate_floating_text`). A streak x2+ shows a
  "STREAK xN" banner.
- The extra points must flow into the existing `Score` / `score_value` so the
  HUD and high score reflect them; keep `score_value` unit-testable.
