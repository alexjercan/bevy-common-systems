Build Bevy games faster

bevy_common_systems is a collection of copy-pastable Bevy utilities - the gameplay components, systems and plugins that almost every game ends up needing, so a new project does not have to rewrite them.

Read the docs Play the examples
cargo add bevy_common_systems --git https://github.com/alexjercan/bevy-common-systems

Modules

What's inside

Each module is one small, game-agnostic concern with an obvious API. Most add runtime behaviour through a single *Plugin; pure-utility modules export plain types and functions. Every card links to its handbook page.

Highlights

A few things it does well

01

One shape for every module

Add a *Plugin, spawn its config component, write an *Input each frame, and read an *Output (or direct Transform writes). Learn one module and you know them all.

Module conventions → 
use bevy::prelude::*;
use bevy_common_systems::prelude::*;

App::new()
    .add_plugins(DefaultPlugins)
    .add_plugins(HealthPlugin)
    .run();
02

Procedural meshes and juice

Build octahedron spheres and noise planets with TriangleMeshBuilder, slice any mesh into flying fragments, then sell the hit with a material flash and a screen vignette.

The mesh module → 
let mesh = TriangleMeshBuilder::octahedron(1.0)
    .subdivide(3)
    .displace_noise(&fbm, 0.5)
    .build();
03

A modding event bus

A generic, serde-friendly event bus for modding and scripting: JSON-authored handlers react to typed events whose payloads travel as serde_json::Value. One example game is built entirely on it.

The modding module → 
// a handler, authored as data
{
  "on": "ReactorTick",
  "do": [
    { "action": "AddEnergy", "amount": 1 }
  ]
}
