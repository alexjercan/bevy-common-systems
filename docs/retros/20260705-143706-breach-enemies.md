# Retro: 14_breach enemy archetypes

- TASK: 20260705-132240
- BRANCH: feat/breach-enemies (squash-merged to master as 36fb239)
- REVIEW ROUNDS: 1 (APPROVE)

Part of the breach fun-pass flow (spike
`docs/spikes/20260705-132024-breach-fun-pass.md`).

## What went well

- The spike's "in-file table, not a SpecCatalog<T>" call (from the bastion
  data-catalog spike) made this a small, self-contained change: an `EnemyKind` enum
  with a `stats()` match and two pure weighting functions, no new type machinery.
- Modelling the mix as pure `archetype_weights(wave) -> [f32;3]` +
  `pick_archetype(wave, roll)` made the interesting behaviour (grunts early, rushers
  from wave 1, brutes from wave 3, share grows with wave, weights sum to 1) fully
  unit-testable off the ECS -- the third task in a row where isolating the pure logic
  is what made it verifiable. The autopilot only reaches wave 0, so without those tests
  the variety would have been unverified.
- Making melee sum per-enemy `dps` (instead of `attackers * ENEMY_DPS`) was a one-line
  change that gave brutes real weight and kept the existing attacker-count juice gate.

## What went wrong

- Nothing broke. The one thing to stay alert to: adding a field to `Enemy` (`dps`)
  meant the test-helper spawn had to be updated too -- a compile error caught it
  immediately, but it is the same "changing a struct ripples into the example's own
  test spawns" pattern that a bare `cargo build` would have hidden (only
  `--all-targets` / `test --example` compiles the tests).

## What to improve next time

- When the autopilot can only exercise a slice of a mechanic (here, wave-0 grunts
  only), say so explicitly in the review and lean on pure tests for the rest, rather
  than implying the headless run proved the variety. Did that here; keep doing it.
- A ranged / exploder enemy was deliberately deferred (needs projectiles or a
  self-destruct AoE the arena has never had). If a later task wants it, it is its own
  spike -- do not fold a new damage modality into a stats-table change.

## Action items

- None blocking. `spawn_wave` still builds a mesh per enemy; if enemy counts grow large
  enough to matter, cache one mesh per archetype.
