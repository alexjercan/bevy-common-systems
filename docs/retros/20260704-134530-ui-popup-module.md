## What went well

- Reading all three hand-rolled copies (06/07/08) before designing settled the
  spike's open question by evidence, not preference: every game already uses a
  screen-space UI text node, so worldspace 3D text was never the right call.
  Same "study all the copies first" move that paid off for camera/shake.
- Keeping the world-to-screen projection in the caller (not taking a camera
  handle in the module) kept the module decoupled and made the refactor a clean
  delete-and-replace. The reviewer independently confirmed no behavior change.
- The component + bundle-builder + custom-layout-escape-hatch shape handled both
  the common "+N" and 07's oddball centered STREAK banner without a rigid API --
  the banner just hand-builds a `Popup` on its own `Node`.
- The concurrent master move (a touch-controls commit landed on master mid-task)
  was handled without a hitch this time: I branched the worktree from local HEAD
  and checked `merge-base --is-ancestor` before merging. The process fix from the
  previous cycle's retro worked as intended.

## What went wrong

- R1.1 (MINOR): I wrote a doc comment claiming "an expired popup is despawned
  even if it lacks Node or TextColor" (the reason I made the animate query
  Option those components) but only tested the *with*-Node path. That is exactly
  the "aspirational comment, test only the easy half" antipattern AGENTS.md
  flags -- and it is the same class of gap as camera/shake's R1.2 last cycle.
  Root cause: I designed the robustness (Option queries) deliberately, then
  tested the common path and treated the edge as self-evidently correct. Writing
  the claim in prose is not testing it.

## What to improve next time

- Concrete tactic for the recurring "test the claim you wrote" gap: before
  finalizing tests, grep my own new doc/comment text for claim words -- "even
  if", "still", "regardless", "always", "never" -- and make sure each such
  claim has an assertion. A prose guarantee about an edge case is a test
  request, not a substitute for one. (This lesson has now recurred across
  overload, modding-json-registry, camera/shake and ui/popup; it is already a
  rule in AGENTS.md, so the failure is procedural, not knowledge -- hence the
  mechanical grep tactic rather than another "be careful" note.)
