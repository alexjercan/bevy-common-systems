## What this is

A batch of gameplay/polish upgrades to the `14_breach` FPS example, seeded by the
fun-pass spike after it found that several of the original asks (menu, mobile,
sounds) already existed, so the real gaps were combos, pickups, enemy variety and
juice. Each landed as its own `/flow` task (implement -> review -> squash-merge ->
retro); the per-task retros in `docs/retros/2026070*-breach-*.md` hold the details.

- **Combos** (`scoring/streak` + `ui/popup`): kills chained inside a window
  multiply the points they are worth; a "+N" floats up per kill and a "COMBO xN +P"
  tally flashes on lapse. Score is now points (persisted), kills kept for the readout.
- **Pickups**: slain enemies may drop a glowing HP / speed / fire-rate pickup; the
  effect is a pure `apply_pickup`, buffs decay on a timer with a HUD readout.
- **Enemy archetypes**: Grunt / Rusher / Brute via an in-file `EnemyKind::stats`
  table and a pure wave-weighted spawn; melee sums per-enemy dps.
- **Juice**: crosshair hit marker, muzzle flash, enemy spawn tells, and a near-death
  red vignette throb.
- **Sounds** (this note's focus, below).

## Sound pass

Breach originally borrowed shared placeholder sines (a gunshot mapped to the fruit
`launch`, a kill to the `combo` chime), which read wrong. This pass gives it its own
combat cues, prefixed `breach_`, generated by `scripts/gen-placeholder-sounds.py`.

The generator gained a `render_fx(kind, f0, f1, dur, amp)` alongside the existing
pure-sine `render`: it produces **noise bursts** (gun, kill) and **pitch sweeps**
(hit, hurt, wave, pickup) with a fast quadratic decay so they read as impacts, not
musical blips. The noise RNG is seeded per file name, so regenerating is
byte-identical (no spurious asset churn).

Six files: `breach_shoot` (dry pop), `breach_hit` (short downward tick),
`breach_kill` (noise burst), `breach_hurt` (low groan), `breach_wave` (rising
alert), `breach_pickup` (bright rise). The `Sfx` table in `examples/14_breach.rs`
maps to these; `menu_select` and `game_over` stay shared, since those are
genre-neutral UI cues and the crate's convention is to share generic sounds rather
than duplicate them.

These are still placeholders: replace any `assets/sounds/breach_*.wav` with real
audio at the same path, no code change needed (see `assets/sounds/README.md`). The
wasm build already ships them -- `web/games/14_breach/index.html` copies the whole
`assets/sounds` dir, so new files are picked up automatically.

## Verification note

Audio can only be confirmed by running (the headless autopilot loads and fires all
combat cues without asset errors, which proves the files exist and are wired; the
actual timbre is a listen-test for the developer). Visual juice that only appears on
input (hit marker, muzzle flash) is not provable by a state-entry screenshot and the
headless framebuffer capture came back black here, so those rest on the no-B0004
autopilot run plus the pure unit tests -- see the juice retro
`docs/retros/20260705-145227-breach-juice.md`.
